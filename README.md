## About The Project

The **Data Space Portal** is an open-source solution for managing dataspaces based on the Eclipse Dataspace Connector (EDC), the Dataspace Protocol (DSP) and the awesome [sovity GmbH](https://sovity.de) base portal. It provides a comprehensive administration interface where organizations can join, manage connectors, and participate in data exchanges.

This repository produces **neutral Docker images** without any specific branding. Images are published to GitHub Container Registry (GHCR):

| Image | Description |
|-------|-------------|
| `ghcr.io/<owner>/dataspace-portal-backend` | Quarkus API backend |
| `ghcr.io/<owner>/dataspace-portal-frontend` | Angular frontend (neutral `default` profile) |
| `ghcr.io/<owner>/dataspace-portal-crawler` | Catalog crawler |
| `ghcr.io/<owner>/dataspace-portal-keycloak` | Keycloak with `portal-theme` and dev realm |

### Runtime Branding (env vars)

The frontend supports runtime brand customization via environment variables:

| Variable | Purpose |
|----------|---------|
| `AUTHORITY_PORTAL_FRONTEND_BRAND_LOGO_URL` | Main logo URL (absolute or relative) |
| `AUTHORITY_PORTAL_FRONTEND_BRAND_LOGO_SMALL_URL` | Small/collapsed logo |
| `AUTHORITY_PORTAL_FRONTEND_BRAND_LOGO_LOGIN_URL` | Login page logo |
| `AUTHORITY_PORTAL_FRONTEND_BRAND_COPYRIGHT` | Copyright text in footer |
| `AUTHORITY_PORTAL_FRONTEND_PORTAL_DISPLAY_NAME` | Portal name shown in UI |
| `AUTHORITY_PORTAL_FRONTEND_DATASPACE_SHORT_NAME` | Dataspace short name |

## Development

To develop new features, once the API definitions are implemented
in the backend, the development of both the frontend and backend can start in parallel.

The frontend has a type-safe fake backend which also uses the API models via an autogenerated TypeScript API Client
Library.

### Local Development

The backend and the frontend can be developed separately:

- [Local Backend Development](authority-portal-backend/README.md#development)
- [Local Frontend Development](authority-portal-frontend/README.md#development)

### Local E2E Development

Local E2E Development is what you use to integration test the backend and frontend locally. Requires JDK21, a Docker
Environment and Node 24.

```shell
# Build + Run Backend
(cd authority-portal-backend && ./gradlew clean quarkusDev)

# Build + Run Frontend (basic version with fake Backend)
(cd authority-portal-frontend && npm i && npm run cold-start-e2e-dev)
```

For more details on how to launch the frontend,
see [Frontend Local E2E Development](authority-portal-frontend/README.md#local-e2e-development).
